# pvl-l5 Home Manager Neovim Boot Failure 2026-06

After deploying `pvl-l5` to
`/nix/store/ikhqnz5bicmf00xc9qclmf3kpb3kvzkh-nixos-system-pvl-l5-26.05.20260611.a037402`
and rebooting, the only failed systemd unit was `home-manager-pvl.service`.

The Home Manager journal showed:

- `mkdir: cannot create directory '/home/pvl/.config/nvim': File exists`
- `ln: failed to create symbolic link '/home/pvl/.config/nvim/init.lua': No such file or directory`

The live state had `/home/pvl/.config/nvim` as a symlink to
`/home/pvl/dotfiles/nvim/.config/nvim`, but `/home/pvl/dotfiles` did not exist
on the host. The generation also contained a managed `.config/nvim/init.lua`,
generated by `programs.neovim` for provider setup. Because the custom activation
linked `.config/nvim` before Home Manager's `linkGeneration`, Home Manager then
tried to create a managed file below that symlink and failed.

The first fix made the editable symlink activation non-fatal. The later cleanup
removed that activation path entirely and lets Home Manager own the generated
Neovim init file:

- remove `home.activation.linkEditableNvim`;
- install `pkgs.vimPlugins.lazy-nvim` through `programs.neovim.plugins`;
- set `xdg.configFile."nvim/init.lua"` with `text = lib.mkDefault ""` and
  `force = true` so Home Manager has a valid managed source even when the Neovim
  module has no generated Lua text, while still letting a real generated Neovim
  config override the default.

Validation:

- `nix build --no-link .#nixosConfigurations.pvl-l5.config.home-manager.users.pvl.home.activationPackage`
- `nix build --no-link --print-out-paths .#nixosConfigurations.pvl-l5.config.home-manager.users.pvl.home.activationPackage`
- checking the fixed generation's `home-files` tree showed a Home
  Manager-managed `.config/nvim/init.lua` under a real `.config/nvim` directory,
  not below an editable dotfiles symlink.

Other current-boot errors observed during triage were not systemd failures:
early ACPI firmware namespace errors, repeated
`i2c_designware AMDI0010:03: controller timed out`, and NVIDIA EDID warnings for
`DP-4`. AMD and NVIDIA DRM drivers both initialized.

## 2026-06-24 Neovim module expansion

The user Neovim module now keeps the full Neovim plugin and Lua configuration
inside `programs.neovim`, rather than restoring an editable dotfiles symlink.
The `xdg.configFile."nvim/init.lua"` default source remains as a fallback so
Home Manager still owns a real generated file path if the Neovim module ever has
no generated Lua payload.

Validation:

- `alejandra --check users/pvl/neovim/default.nix`
- `statix check -- users/pvl/neovim/default.nix`
- `deadnix users/pvl/neovim/default.nix`
- `nix build --no-link .#nixosConfigurations.pvl-l5.config.home-manager.users.pvl.home.activationPackage`
