# Incus Machines Module

**Date**: 2026-03-24

## Summary

Canonical summary of the reusable Incus host module and its March 2026 follow-up
hardening: helper extraction, image handling, recreate triggers, GC semantics,
and recovery rules for managed guests declared under `services.incusMachines`.

## Key Decisions

- **Device submodule is flat**: `source`, `path`, `shift`, `pool`,
  `extraProperties` as top-level fields. `type` defaults to `"disk"`.
- **Volume vs host mount heuristic**: `source` starting with `/` is a host path;
  otherwise it's a volume name (includes `pool` property).
- **`/dev` host-path mounts are special**: host-path disk devices under `/dev`
  are treated as existing device trees, not persistent source directories. The
  module does not tmpfiles-create them and GC never removes them.
- **No stateDir**: persistent state is just a regular disk device entry with
  `removalPolicy = "delete"` to opt into cleanup.
- **Config changes trigger recreate**: config keys are hashed and stored as
  `user.config-hash`. Mismatch triggers stop+delete+create.
- **Disk devices sync in-place**: disk devices are reconciled via
  add/set/remove/unset without recreating the container.
- **Non-disk devices are create-only**: `gpu`, `unix-char`, and similar devices
  are part of the recreate hash and change via stop+delete+create.
- **bootTag / recreateTag**: declarative knobs to force restart or recreate on
  next rebuild by bumping a string value.
- **imageTag**: top-level declarative knob to force re-import of the shared
  `local:nixos-incus-base` alias even when the underlying image build output
  path has not changed.
- **Image refresh versus recreate**: rebuilt local image store paths must not by
  themselves trigger guest recreate; image refresh is governed by `imageTag`,
  while full guest recreate remains gated by explicit config changes or
  `recreateTag`.
- **GC service**: `incus-machines-gc` removes containers tagged
  `user.managed-by=nixos` that are no longer declared. Respects per-container
  `removalPolicy` (stop-only, delete-container, delete-all) and per-device
  `removalPolicy` (keep, delete) for source dir cleanup.
- **Base image service**: auto-generated by the module when any machines are
  defined, eliminating duplication across hosts.
- **Module path**: the canonical module path is `lib/incus/default.nix`. Hosts
  should keep importing the directory path `../../lib/incus`; the old
  `lib/incus.nix` forwarding shim is obsolete.
- **Helper split**: shell-heavy runtime logic lives in `lib/incus/helper.sh`,
  with wrapper commands preserved for reconcile and settlement flows.
- **Structured environment payloads**: systemd `Environment=` values carrying
  JSON must be shell-escaped as whole assignments so the helper receives valid
  structured input.
- **Structured reconcile input**: config/meta writes stay JSON-driven instead of
  sourced shell fragments, and shell argument assembly for
  `incus config device add` uses arrays rather than unquoted scalar expansion.
- **Per-machine image model**: each machine may override `image` and
  `imageAlias`; `imageTag` refreshes every declared alias, and duplicate alias
  reuse for different image sources is rejected.
- **Image correctness boundary**: `incus-images.service` is a real hard
  prerequisite for guest create/recreate, missing aliases are recovered from
  existing local image objects via `user.base-image-id`, and the guest create
  path performs a just-in-time image preflight for its exact declared alias.
- **Image and GC units are rerunnable**: image import and GC are ordinary
  idempotent oneshots, not `RemainAfterExit` memoized success markers.
- **Preseed epoch knob**: `services.incusMachines.preseedTag` is the explicit
  operator bump for disruptive parent-host Incus fabric changes that should
  force guest recreate.
- **GC timing**: `incus-machines-gc.service` is switch-scoped host maintenance,
  not a per-guest prerequisite, and reruns when Incus-related declaration state
  changes.
- **GC metadata sync**: instance `removalPolicy` and per-disk cleanup metadata
  are reconciled in place on existing guests so policy changes apply without a
  recreate.
- **GC safety**: dangerous `delete-all` source-path cleanup fails closed, and GC
  must fail if Incus listing fails instead of treating query failure as an empty
  result.
- **Recoverable broken instances**: a guest whose instance record exists but its
  rootfs/storage is missing is force-deleted and recreated once per helper run
  when Incus reports known broken-instance start errors.
- **Evaluation guardrails**: duplicate static `ipv4Address` declarations are an
  evaluation error.

## Source of Truth

- `lib/incus/default.nix` — module definition
- `lib/incus/helper.sh` — lifecycle helper implementation
- `hosts/<parent-incus-host>/incus.nix` — parent-host machine declarations +
  preseed
- `hosts/<nested-incus-host>/incus.nix` — nested-host machine declarations +
  preseed
